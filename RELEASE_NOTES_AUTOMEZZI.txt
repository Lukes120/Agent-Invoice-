PATTERN ARCHITETTURALE
================================================================================

Pattern consume-POL multi-line con riscrittura totale POL libere generic
"TEST €1" pre-create da Acquisti su ogni OdA-ledger annuale.

Per ogni riga XML della fattura, l'agent:
  1. Identifica voce (locazione/servizi/tassa) dalla descrizione
  2. Estrae targa veicolo (regex sulla desc + fallback CodiceArticolo)
  3. Estrae numero contratto (DatiContratto.IdDocumento o regex desc)
  4. Lookup in PARCO_BY_TARGA / PARCO_BY_CONTRATTO →
     classificazione (POOL/uso_promiscuo/super_lusso)
  5. Conto contabile = CONTO_AUTOMEZZI[(voce, classificazione)]
  6. Tax_id = TAX_AUTOMEZZI[vat][(voce, classificazione)]
  7. Prende prima POL libera FIFO sull'OdA target
  8. Riscrive completamente la POL: name = desc XML originale,
     price = importo XML, taxes = tax target, qty_received=1
  9. Crea move_line con purchase_line_id collegato

Routing OdA:
  - Multi-contratto (Tecnoalt, UnipolRental): mapping_entry.contratti
    + fallback oda_default
  - Singolo (altri): mapping_entry.oda_fisso

PROCEDURA DEPLOY
================================================================================

1. Backup precauzionale:
     Copy-Item -Path "C:\odoo_apps\invoice_agent" `
               -Destination "C:\odoo_apps\invoice_agent_backup_20260508_preautomezzi" `
               -Recurse

2. Fermare il webapp.

3. Estrarre lo zip:
     $ProgressPreference = 'SilentlyContinue'
     Expand-Archive -Path "$env:USERPROFILE\Desktop\deploy_server.zip" `
                    -DestinationPath "C:\odoo_apps\invoice_agent\" -Force

4. NESSUNA nuova dipendenza pip da installare.

5. NESSUNA migration DB (extra_po_lines_json già presente).

6. Avviare webapp:
     python "Agent Invoices/app.py"

7. Test reale: lancia una run dell'agent (o aspetta arrivo nuove fatture
   automezzi via SDI), poi vai sul filtro `MAPPATURA_AUTOMEZZI`:
   - Apri 1 fattura per fornitore (idealmente piccola)
   - Click "Crea bozza Automezzi" sul box verde
   - Verifica su Odoo: bozza creata, POL aggiornata, purchase_line_id
     collegato, attachment registered=True

8. Bulk: una volta validato, click "Crea bozze Automezzi" sulla pagina
   `/invoices?class=MAPPATURA_AUTOMEZZI`.

LIMITAZIONI NOTE
================================================================================

1. Tecnoalt: targa veicolo è in <CodiceArticolo><TARGA>, non in
   descrizione. L'agent attualmente la cerca solo nella descrizione
   → fallback "uso_promiscuo" conservativo. Le fatture Tecnoalt vengono
   create correttamente ma classificate sempre uso_promiscuo invece
   di POOL. Da migliorare in iterazione successiva.

2. UnipolRental multi-origin (es. fatture con "P03495,P03365"): il writer
   raggruppa per OdA target dedotto dal numero contratto della singola
   riga. Da verificare empiricamente sulla prima fattura test.

3. ALD: il numero contratto in ALD XML è in <DatiContratto> con
   <RiferimentoNumeroLinea>. Il writer attuale prende il primo contratto
   trovato come fallback per tutte le righe, NON per linea. Funziona
   se la fattura ha 1 contratto principale (caso comune ALD).

4. LeasePlan: nessun OdA attivo, auto_write=False. Quando arriverà la
   prima fattura, Acquisti dovrà aprire un OdA + ~45 POL TEST per 9 mesi.
