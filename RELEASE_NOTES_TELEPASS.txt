CASI EDGE NOTI
================================================================================

1. Fattura con N>1 righe XML: l'agent prende N POL libere consecutive
   (FIFO id ascending) e le riscrive ognuna. Se le POL libere su P03722
   sono <N, errore esplicito "POL libere insufficienti".

2. Voce non riconosciuta dalla descrizione XML: di default cade in
   'canone' (conto 420840 70%). Casi noti: tutte le voci principali
   sono coperte (Canone, Parcheggi, Bollo, Quota associativa).

3. Aliquota XML diversa da 22/0: di default usa tax_id 11 (22%).
   Edge case raro.

4. cc cliente non in mappa (es. cc Utterson 184015247 storico): il
   resolve_mapping_entry torna None, l'endpoint risponde 400 con
   "Codice cliente NNN non mappato per Telepass". Aggiungere il cc
   in config/rules.py se serve.
